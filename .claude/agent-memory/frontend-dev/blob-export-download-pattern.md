For server-generated exports (not the client-side CSV in overtime-report), the service method returns `Observable<HttpResponse<Blob>>` using `http.get(url, { responseType: 'blob', observe: 'response', withCredentials: true })`. The component reads `resp.body` and derives the filename from the `Content-Disposition` header (falling back to a constructed `name-{month}.{format}`), then does the standard `URL.createObjectURL` → anchor `.click()` → `revokeObjectURL` download.

**Why:** US-ATT-007 export (AC-4) needs CSV/Excel/PDF generated by the backend (ClosedXML/QuestPDF), so the FE just downloads a blob — it can't build xlsx/pdf client-side like the overtime-report CSV.

**How to apply:**
- Service spec (HttpTestingController): assert `req.request.responseType === 'blob'` and `req.flush(new Blob([...]))`.
- Component spec (mocked service): return `of(new HttpResponse<Blob>({ body: blob, headers: new HttpHeaders({ 'Content-Disposition': 'attachment; filename="x.csv"' }) }))`, then spy on `URL.createObjectURL`/`revokeObjectURL` and stub `document.createElement` to return `{ href:'', download:'', click: spy }` — same pattern as [[jasmine-optional-arg-spy]] sibling overtime-report CSV test.
