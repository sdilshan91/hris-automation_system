# US-RPT-004: Export Reports to CSV / PDF / Excel

## 1. Description
**As an** HR Officer,
**I want to** export any generated report to CSV, PDF, or Excel (.xlsx) format,
**So that** I can share reports with stakeholders who do not have system access, archive reports offline, and submit statutory reports in required formats.

## 2. Preconditions
- The HR Officer is authenticated and has `Reports.Export` permission within their tenant.
- A report has been generated (any report from US-RPT-001, US-RPT-002, or US-RPT-003).
- The Reporting module is enabled for the tenant's subscription plan.

## 3. Acceptance Criteria (IEEE 830 S3.2 - Specific Requirements)
| # | Given | When | Then |
|---|-------|------|------|
| AC-1 | HR Officer has generated a Headcount Summary report | They click the "Export" button | A dropdown appears with format options: CSV, Excel (.xlsx), and PDF. Selecting a format initiates the export. |
| AC-2 | HR Officer selects "Excel" for a report with 500 rows | The export is generated | A Hangfire background job generates the .xlsx file using ClosedXML, including: a header with report title, filters applied, and generation timestamp; formatted data table with column headers; and a chart sheet (if applicable). The user receives an in-app notification (SignalR) when the file is ready, with a download link. |
| AC-3 | HR Officer selects "PDF" for a report | The export is generated | A Hangfire background job generates a PDF using QuestPDF, including: tenant logo and branding, report title, filters, charts rendered as images, data tables, and page numbers. The file is available via a signed URL. |
| AC-4 | HR Officer selects "CSV" for a simple data report | The export completes | A CSV file is generated (small reports inline, large reports async via Hangfire) with UTF-8 BOM encoding, comma-separated values, headers in the first row, and proper escaping of special characters. |
| AC-5 | An export is triggered in Tenant A | A user from Tenant B attempts to download the file | The signed URL is tenant-scoped; Tenant B users receive a 403 Forbidden error. The file is stored in tenant-isolated object storage. |

## 4. Functional Requirements (IEEE 830 S3.2)
- FR-1: The system SHALL support export to three formats: CSV, Excel (.xlsx), and PDF.
- FR-2: CSV export SHALL use UTF-8 BOM encoding, comma separator, and RFC 4180 compliant escaping.
- FR-3: Excel export SHALL use ClosedXML (free/open-source) to generate .xlsx files with formatted headers, data validation, and optional chart sheets.
- FR-4: PDF export SHALL use QuestPDF (free/open-source) to generate branded PDFs with tenant logo, report metadata, charts (rendered as PNG images), data tables, and pagination.
- FR-5: Small exports (< 1000 rows) MAY be generated synchronously; large exports (>= 1000 rows) SHALL be generated asynchronously via Hangfire.
- FR-6: The system SHALL store generated export files in tenant-isolated object storage at `{tenantId}/exports/{reportType}/{yyyy}/{mm}/{filename}`.
- FR-7: The system SHALL provide a signed download URL with a configurable expiry (default: 15 minutes).
- FR-8: The system SHALL notify the user via SignalR when an async export is ready, with the download link.
- FR-9: The system SHALL audit every export action, logging the report type, filters, row count, format, and actor.
- FR-10: The system SHALL limit concurrent exports per user to 3 to prevent abuse.

## 5. Non-Functional Requirements (IEEE 830 S3.3)
- NFR-1: Synchronous CSV export (< 1000 rows) SHALL complete within 2 seconds.
- NFR-2: Asynchronous exports SHALL complete within 60 seconds for up to 50,000 rows.
- NFR-3: Export files SHALL be stored in tenant-isolated paths; cross-tenant access SHALL return 403.
- NFR-4: Signed URLs SHALL expire within 15 minutes (configurable).
- NFR-5: The export button and format selector SHALL be fully responsive and accessible (WCAG 2.1 AA).
- NFR-6: Export operations SHALL not degrade the performance of interactive report viewing (use background jobs).
- NFR-7: All export data SHALL be isolated by tenant via PostgreSQL RLS.

## 6. Business Rules
- BR-1: Export respects the same data scoping as the report view (permission-based: `View.All` vs. `View.Team`).
- BR-2: Sensitive data (bank accounts, national IDs) is included in exports only if the user has `ViewSensitive` permission; otherwise, masked values are exported.
- BR-3: Export files are retained for 7 days, then automatically purged by a Hangfire cleanup job.
- BR-4: A maximum of 100,000 rows can be exported in a single operation.
- BR-5: PDF exports include a "Generated by [Tenant Name] via HRM SaaS" footer with timestamp.
- BR-6: Excel exports include a "Filters Applied" section in the header showing which filters were used.

## 7. Data Requirements
**Export request:**
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| report_type | varchar(50) | Yes | Valid report key |
| format | varchar(10) | Yes | "csv", "xlsx", or "pdf" |
| filters | jsonb | Yes | Same filters used for report generation |
| include_charts | boolean | No | Default: true (for PDF and Excel) |

**Output:** Export job status with download URL (when complete).

## 8. UI/UX Notes
- Export button: positioned in the report toolbar (top right), with a dropdown arrow showing format options.
- Each format option shows a file type icon (CSV icon, Excel green icon, PDF red icon).
- For synchronous exports: browser initiates a direct download.
- For async exports: a progress indicator appears ("Generating export..."), replaced by a download button when ready.
- Export history: accessible from a "My Exports" section showing recent exports with status, format, size, and download link.
- On mobile (< 768px): export button is accessible via an overflow menu (three-dot icon).
- Toast notification on export completion: "Your [format] export is ready. [Download]" with a clickable download link.

## 9. Dependencies
- US-RPT-001, US-RPT-002, US-RPT-003: Reports must be generated before export.
- US-NTF-001: In-app notification for async export completion.
- Hangfire: For async export generation.
- File & Document Management (Technical Doc S26): For export file storage and signed URLs.
- US-NTF-004: Audit trail for export actions.

## 10. Assumptions & Constraints
- ClosedXML (free/open-source, MIT license) is used for Excel generation; no dependency on Microsoft Office.
- QuestPDF (free/open-source) is used for PDF generation.
- Chart images for PDF exports are rendered server-side (e.g., using SkiaSharp or a headless browser capture).
- Object storage (Azure Blob / S3 / MinIO) is available for export file storage.
- Only free/open-source libraries are used.
- The system uses PostgreSQL with RLS as defense-in-depth for tenant isolation.

## 11. Test Hints
- **CSV export:** Generate a headcount report with 100 employees; export to CSV; verify file encoding (UTF-8 BOM), header row, and data accuracy.
- **Excel export:** Export to .xlsx; open in a spreadsheet application; verify formatted headers, data, and filter metadata.
- **PDF export:** Export to PDF; verify tenant logo, charts, data tables, pagination, and footer branding.
- **Async export:** Generate a report with 5000 rows; trigger export; verify Hangfire job is created and notification sent on completion.
- **Signed URL expiry:** Generate an export; wait 16 minutes; attempt download; verify 403 or link expired response.
- **Tenant isolation:** Generate exports in Tenant A; attempt to access the file URL from Tenant B; verify 403 Forbidden.
- **Sensitive data masking:** Export bank advice without `ViewSensitive` permission; verify account numbers are masked in the export file.
- **Concurrent limit:** Trigger 4 exports simultaneously; verify the 4th is queued or rejected with a message.
- **Audit trail:** Trigger an export; verify audit record captures report type, format, row count, and actor.
- **File cleanup:** Verify Hangfire job purges export files older than 7 days.
