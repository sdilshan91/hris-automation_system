# TC-AUTH-011: Reset password with valid token works

## 1. Test Objective
Verify that a user can successfully reset their password using a valid reset token, and that all existing refresh tokens are revoked, failed login counter is reset, and the lockout is cleared.

## 2. Related Requirements
- User Story: US-AUTH-004
- Acceptance Criteria: AC-3
- Functional Requirements: FR-2, FR-5, FR-6, FR-8
- Business Rules: BR-4

## 3. Preconditions
- User `john@acme.com` has requested a password reset and received a valid reset token.
- The reset token has not expired (within the 1-hour window).
- The user may have existing refresh tokens in one or more tenants.

## 4. Test Data
| Field | Value | Notes |
|-------|-------|-------|
| Email | john@acme.com | User with valid reset token |
| Token | (valid reset token from email) | Generated by Identity token provider |
| New Password | N3wS3cure!Pass2026 | Meets tenant password policy |
| Old Password | S3cure!Pass2026 | Previous password |
| Reset endpoint | POST /api/v1/auth/reset-password | Accepts { email, token, newPassword } |

## 5. Test Steps
| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Navigate to the reset link from the email: `https://acme.yourhrm.com/reset-password?token={token}&email=john@acme.com` | Reset password page is displayed with new password and confirm password fields, and a password strength indicator. |
| 2 | Enter a new password `N3wS3cure!Pass2026` in both fields | Password strength indicator shows strong; policy checklist items are all checked. |
| 3 | Submit the form, which calls `POST /api/v1/auth/reset-password` with `{ email: "john@acme.com", token: "{token}", newPassword: "N3wS3cure!Pass2026" }` | HTTP 200 OK with message "Password updated successfully." |
| 4 | Verify the `password_hash` has been updated in the `users` table | New hash differs from the old hash and validates against the new password. |
| 5 | Verify `password_changed_at` is updated to the current timestamp | Timestamp is recent (within seconds of the request). |
| 6 | Verify `failed_login_count` is reset to 0 | Counter is cleared. |
| 7 | Verify `locked_until` is set to null | Any existing lockout is cleared. |
| 8 | Verify all existing refresh tokens for this user across ALL tenants are revoked | All `refresh_token` records for this user_id have `revoked_at` set. |
| 9 | Verify a `password_reset_completed` audit event is logged | Audit record contains user_id, tenant_id, IP address. |
| 10 | Attempt to log in with the old password `S3cure!Pass2026` | HTTP 401 Unauthorized; old password no longer works. |
| 11 | Log in with the new password `N3wS3cure!Pass2026` | HTTP 200; login succeeds with new tokens issued. |
| 12 | Verify the UI displays "Password updated successfully. Redirecting to login..." with auto-redirect after 3 seconds | User is smoothly redirected to the login page. |

## 6. Postconditions
- The user's password has been updated.
- All previous refresh tokens are revoked across all tenants.
- The user can log in only with the new password.
- Lockout state is cleared.
