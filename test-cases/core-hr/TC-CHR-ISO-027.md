# TC-CHR-ISO-027: RLS blocks direct DB queries across tenants for locations

## 1. Test Objective
Verify that PostgreSQL Row-Level Security (via EF Core global query filters) prevents direct database queries from returning location records belonging to a different tenant. Even if an application bug bypassed the service layer, the database-level filter would block cross-tenant data access. This validates FR-8 and NFR-2.

## 2. Related Requirements
- User Story: US-CHR-007
- Functional Requirements: FR-8
- Non-Functional Requirements: NFR-2

## 3. Preconditions
- Tenant "acme" (tenant_id = UUID-A) exists with locations: "Colombo Office" (location_id = LOC-A1).
- Tenant "globex" (tenant_id = UUID-B) exists with locations: "New York HQ" (location_id = LOC-B1).
- Access to the EF Core DbContext or direct database connection for testing the query filter behavior.

## 4. Test Data
| Field | Value | Notes |
|-------|-------|-------|
| Tenant A ID | UUID-A | acme |
| Tenant B ID | UUID-B | globex |
| Location A1 | LOC-A1 (Colombo Office) | Belongs to acme |
| Location B1 | LOC-B1 (New York HQ) | Belongs to globex |

## 5. Test Steps
| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Set the ITenantContext to tenant "acme" (UUID-A) | Tenant context is resolved to acme. |
| 2 | Execute `dbContext.Locations.ToListAsync()` | Only locations with `tenant_id = UUID-A` are returned. "Colombo Office" is present. "New York HQ" is NOT present. |
| 3 | Execute `dbContext.Locations.FindAsync(LOC-B1)` (globex's location ID) | Returns `null`. The global query filter prevents access to globex's location even by direct ID lookup. |
| 4 | Execute `dbContext.Locations.IgnoreQueryFilters().Where(l => l.TenantId == UUID-B).ToListAsync()` | This should only be allowed in specific service-level contexts (e.g., system admin). In normal application code, `IgnoreQueryFilters()` is not used for location queries. |
| 5 | Set the ITenantContext to tenant "globex" (UUID-B) | Tenant context switches to globex. |
| 6 | Execute `dbContext.Locations.ToListAsync()` | Only locations with `tenant_id = UUID-B` are returned. "New York HQ" is present. "Colombo Office" is NOT present. |
| 7 | Verify the SQL generated by EF Core contains a `WHERE tenant_id = @tenantId` clause | The query plan/SQL log shows the tenant filter is applied at the database level. |

## 6. Postconditions
- No cross-tenant location data was accessible through the EF Core query layer.
- Global query filters are confirmed to be active for the Location entity.
