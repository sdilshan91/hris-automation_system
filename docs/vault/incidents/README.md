# Incidents

Bugs, production incidents, post-mortems. The goal is to capture the *why* and the *lesson*, not to re-document the fix (the commit does that).

## When to add an incident

- A bug that survived past initial dev because the failure mode was non-obvious
- A production-affecting issue
- A near-miss that revealed a latent design flaw

Trivial bugs caught in PR review don't belong here.

## Format

`YYYY-MM-DD-<short-slug>.md`. Use [[_template]].
